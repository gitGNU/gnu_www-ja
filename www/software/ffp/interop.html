Free Film Project - Interoperability

This will document all the interoperability requirements for the project. This is intended as a guide for other programmers wishing to contribute to the project.

Requirements

  1. Where applicable, stand-alone software should be able to interoperate with existing GNU software.
    • eg: Project managers should be able to read/write GNU makefiles and GNU autoconfig files.

  2. Rendering packages and modellers should be able to support both native FFP files and Renderman-compliant files.
    • This project will not use, or promote, non-free rendering packages. At the time this project started, no non-free rendering packages existed. This is no longer the case, and all references to such packages, EXCEPT for file formats and APIs, are now revoked and removed. The project may end up creating its own rendering package, as all existing packages use techniques that have been superseded. This is where the freedom issue is important. FFP developers CANNOT enhance non-free software, and we can't produce the sort of quality we want, using software that can't do the work.
