關於如何使用 savannah 進行中譯小組網頁的管理：

1. 您必須使用 cvs ，這通常表示您必須要在一台 GNU/Linux 或其它 
   Unix-like 系統上進行操作。因此先確定在您的系統上是否安裝有 cvs 。

2. 向中譯小組的任何一位管理員請求開放權限，我們馬上就可以開放。（如果
   您不知道要同誰聯繫，直接寄郵件給作者 <chliu@gnu.org> 就可以了。）如
   果您想要幫忙管理，也請告訴我們。

3. 接下來您必須在您的系統上進行一些設定。首先是 ssh ：

>> ssh-keygen

在某些版本中，您也許必須鍵入下行以作為替代：

>> ssh-keygen -t rsa1

ssh 會詢問您 passphrase ，您必須設定一個，不要只是鍵入 newline 字元。
（這個 passphrase 在您每一次進行網站管理的動作時都會詢問您，因此請自行
記好。）然後您必須將“Protocol 1,2”這個字串加入到您的 ssh 組態檔案中，
您可以這樣作到：

>> echo "Protocol 1,2" > ~/.ssh/config

4. 然後您就可以到 savannah 中編輯您的 sshkey 資料了：

http://savannah.gnu.org/account/editsshkeys.php

請注意您的公開鍵資料預設是寫在 ~/.ssh/identity.pub 檔案中的〈如果您沒
有特別給它更動的話〉。 savannah 的 cron 排程會在下一個小時內更新您的資
料。（泡杯茶）

5. 先建立一個目錄用來存放中譯小組的網站，然後移到該目錄鍵入：

>> export CVS_RSH=ssh

以使您的 cvs 使用 ssh 安全認證，再鍵入：

>> cvs -z3 -d<your-savannah-account>@subversions.gnu.org:/webcvs co software/chinese/

其中 <your-savannah-account> 要以您自己的 savannah account 來取代。如
作者的是“chstoneliu”，故鍵入的是：

>> cvs -z3 -dchstoneliu@subversions.gnu.org:/webcvs co software/chinese/

這樣就會將網站中的所有頁面下載到您的機器上。（下載完您就可以 off-line 
了，省一點。）現在您應該是在該目錄下，（作者的是在 ~/gnu/ 。）先移動到
中譯小組的根目錄：

>> cd software/chinese/

比如您現在要編輯 index.html 這個檔案：

>> emacs index.html

（其它編輯器也可以。）東看看西找找，把要更新的部份反應到頁面上。（比如
將某人列為某篇文章的翻譯。）然後退出編輯器。現在您應該還在 
~/gnu/software/chinese/ ，確認更新無誤之後：（如果您剛才 off-line 了，
現在必須再 on-line 。）

>> cvs add index.html

您會被詢問剛才所說的 passphrase 。待您己經將所有的網站工作完成後，鍵入：

>> cvs commit

您也會被詢問 passphrase ，然後會出現一個編輯螢幕，（通常是 vi ，但是您
也可以將預設值修改為 Emacs 。沒什麼必要，因為只是要鍵入個一行文字而已。）
讓您可以說明您此次所作出的修改的簡短解釋，操作例子如下：

a. 鍵入字元“i”，這會使您進入 vi 的輸入模式。
b. 鍵入解釋的文本，如：“Modified: Syue-Ping Ma's suggestion.”。
c. 鍵入 <ESC> 回到 vi 的指令模式。
d. 鍵入“:wq”以儲存並且離開 vi 。

稍待一會，您就可以從螢幕上看到您的修改已經上載到 savannah 的訊息，完成
網站管理的工作。


如何偷懶？
----------

每次都要鍵入一長串實在不太經濟，雖然可以用 paste 的方式解決，但是顯然
不符合我們 hacker 對於操作優雅的要求。因此下面是作者的作法，先將下面這
幾行加入到您的 ~/.bashrc 檔案中：

# >> webmastering gnu www pages...
export CVS_RSH=ssh
alias cvsctt="cvs -z3 -dchstoneliu@subversions.gnu.org:/webcvs co software/chinese/"

# >> big5-gb2312 inter-translating using iconv...
alias b2g="iconv -f big5 -t gb2312 -o "
alias g2b="iconv -f gb2312 -t big5 -o "

但是記得要將 chstoneliu 改成您自已的 savannah account 才是，這樣子只要
鍵入：

>> cvsctt

就可以了。比起原來的

>> cvs -z3 -dchstoneliu@subversions.gnu.org:/webcvs co software/chinese/

好多了。同時因為直接在 ~/.bashrc 中加入“export CVS_RSH=ssh”，所以您
也不必每一次開新的 shell session 都必須鍵入。（因為在 shell 啟動時就已
經代您鍵入了。）


簡繁中文的同時提供。
--------------------

您可以發現上面另外還有 big5-gb2312 兩種編碼的互轉 alias(es) 。以作者維
護 GNU 首頁為例，實際上的編輯檔案有兩個：

home.zh.html
home.zhs.html

這兩者都是以 big5 編碼的，但是其中 home.zhs.html 是採用簡體中文的術語。
（白若玉提供。）待完成更新後，鍵入：

b2g home.cn.html home.zhs.html

這樣子就做出了 gb2312 編碼的簡體中文頁面。（請注意引數輸入的順序，顛倒
將會使 home.zhs.html 成為空白檔案。）因此如果您使用的是 gb2312 編碼的
工作環境，您的工作檔案將有可能是：

file.cn.html
file.zht.html

皆以 gb2312 進行編碼，待兩者更新且同步後，鍵入：

g2b file.zh.html file.zht.html

以製作出相對應的繁體中文頁面。GNU Project 新完成以及更新的中譯頁面都必
須同時提供簡繁版本，否則不進行上載的動作。希望我們的每一位成員都能夠做
到提供兩種版本。（如果您的工作環境並不允許，還是直接將您完成的譯本寄給
作者就可以了。許多人由於外在的客觀條件，一時之間在操作上會有困難，但我
們希望您可以瞭解到這完全不會也不應影響到您對於 GNU 中譯工作的參與。如
果有人曾經藉此攻擊您的工作，請讓我們知道發生了什麼事。）

另外對於簡繁術語的差異，目前遇到的大部份都已經收錄在：

http://www.gnu.org/software/chinese/terms.cn.xml
http://www.gnu.org/software/chinese/terms.zh.xml

您可以自行參閱，而後自然地採用於您的工作上。（當然您也可以將您的新發現
收錄進去，記得把自己列為該文本的 contributor 。）不必太在意這些術語的
差異是否完全且正確地反應出，先將譯本完成，我們再來看如何更進一步地改進
它。

----------------------------------------------------------------------

這裡應該已經提供了足夠的資料可以使您上手了，如果還有問題，寄到 
<chliu@gnu.org> 就可以了。

注意在 cvs commit 之前，請先確定您工作中的版本是目前網站中的最新，以免
有人先您一步作出更新，而您又將他的更新給覆蓋回來了。（因此如果您 
commit 與 cvsctt 之間差了一段時間，最好先確認一下。但若您用 cvsctt 來
保持最新時，記得先將您的工作檔案備份到別的目錄，否則變成您的被蓋掉了。）

基本上只要您表達了希望參與的想法，權限馬上就可以給予。您可以將您正在進
行翻譯的頁面放在其中 efforts 目錄下，（可以作為半公開徵詢公眾意見的地
方。）待完成後通知作者反應到正式的 GNU 網站上。如果您翻譯的是使用手冊，
則可以放到 manual 目錄下進行您的工作。（如果您需要子目錄使用，請先通知
作者。）

其它住在 savannah 中的計畫也是如此用法，所以如果您申請了自己的一個在 
savannah 上，那麼您應該知道要如何管理了…〈源碼的管理方式亦同，只是下
不同參數，請自行參考您所參與的計畫的 cvs 頁面〉。
