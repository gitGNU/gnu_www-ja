Como conseguir ajuda com o Software GNU

[ Catalão | Francês | Inglês | português do Brasil ]

A Fundação para o Software Livre não fornece suporte técnico. Nossa missão é desenvolver software, porque este é o modo mais eficiente de utilizar nosso tempo para que mais atividades possam ser realizadas com software livre Nós deixamos para outros a tarefa de ganhar a vida fornecendo suporte. Nós vemos os programadores como prestadores de serviços, de forma semelhante ao que médicos e advogados fazem hoje; tanto o conhecimento médico quanto jurídico são livremente redistribuídos, mas os profissionais cobram pelo serviço. Por isso, por favor não nos escreva nem telefone para o escritório FSF pedindo por suporte técnico.

Você pode obter a Documentação do Projeto GNU por vários meios, que devem responder à maioria das suas questões.

Nós hospedamos várias listas de e-mail sobre software GNU. As listas cujo nome se inicia por help- servem para pedir ajuda à comunidade. Está disponível também uma relação feita à mão das listas do GNU. Se você não conseguir encontrar uma lista sobre um programa GNU em particular, por favor use a lista help-gnu-utils@gnu.org.

O Diretório de Serviços GNU (58k caracteres) é uma lista de pessoas que oferecem suporte e outros serviços de consultoria. Ele também está no arquivo "etc/SERVICE" na distribuição do GNU Emacs, "SERVICE" na distribuição do GCC, e "/pub/gnu/GNUinfo/SERVICE" no servidor de FTP do GNU. Nos contacte para obter uma cópia ou para ser listado nele.

Se você encontrar uma deficiência em qualquer software GNU, nós gostaríamos de saber. Por isso nós também mantemos muitas listas sobre bugs em software GNU. Mesmo que uma lista não seja listada, em geral o endereço <bug-PROGRAMA@gnu.org> chegará ao local correto para informar bugs (via um redirecionamento de e-mail). Portanto, se você encontrar um bug em uma versão oficial de um programa GNU, por favor tente informa-lo desta forma.

Quando nós recebemos um relatório de bug, nós geralmente tentamos consertar o problema. Embora nossas correções de bugs possam parecer assistência individual, elas não são; elas são parte do trabalho de se preparar uma nova versão aperfeiçoada. Nós podemos enviar para você um patch para o bug de modo que você nos ajude a testar a correção e garantir a sua qualidade. Se o seu relatório de bug não receber uma solução de nós, você ainda pode conseguir uma de outro usuário que leia nossas listas de e-mail sobre bugs. Ou então, utilize o Diretório de Serviços GNU (58k caracteres).

Por favor não nos peça ajuda para a instalação de software ou para aprender como utiliza-lo -- mas nos diga se um script de instalação falhar ou se a documentação não for clara.

Finalmente, observe que atualmente muitas empresas redistribuem software GNU, frequentemente como parte de uma distribuição do GNU/Linux. Quando você encontrar bugs em um programa GNU que você instalou como parte de uma dada distribuição do GNU/Linux, é frequentemente recomendado tentar antes informar o bug diretamente para o distribuidor e não para nós. Às vezes, os distribuidores tem versões modificadas do software GNU (eles são livres para fazer isso!) ou eles estão utilizando versões antigas. Portanto, eles podem ser as melhores pessoas para localizar o bug como pertencendo a uma distribuição em particular.
