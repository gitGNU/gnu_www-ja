Violação da Licença Geral Pública do GNU (GPL) Pelo RTLinux

Boston, Massachusetts, EUA - 14 de Setembro de 2001
A Fundação pelo Software Livre fez um pronunciamento oficial sobre as
violações da Licença Geral Pública do GNU (GPL) pelo RTLinux.
Victor Yodaiken, CEP do FSMLabs (que distribui o RTLinux) utilizou uma patente
para impor termos restritos de distribuição sobre um programa coberto pela GPL.
A FSF se opõe à patentes de software, e além disso acredita que a patente de
Yodaiken viola a GPL para o kernel chamado Linux.

Patentes de software são uma política governamental danosa de criar monopólios
que restringem os usuários de computadores. Nós nos opomos a esta política,
e nós achamos uma vergonha que Victor Yoidaken escolheu obter uma patente
para uma idéia que nós acreditamos que não deveria ser, e não é, patenteável.
A patente cobre o gerenciamento em tempo-real de interrupções utilizando uma
camada de emulação de software para o mascaramente de interrupções, de
modo que as interrupções possam ser priorizadas. Existe significativa
"arte anterior" para isso.

Yodaiken tentou utilizar a patente para impor termos restritivos sobre um
programa coberto pela GPL (Linux, o kernel utilizado no sistema operacional
GNU/Linux). Esses termos conflitam com a Licença Geral Pública do GNU,
e impo-los é uma violação da GPL. Ele, como todo mundo, tem a responsabilidade
de obedecer aos termos da GPL ou então encerrar a sua distribuição que viola
a GPL. Qualquer um que redistribui uma versão modificada do Linux sob a
patente restritiva que Yodaiken utiliza também está violando a GPL.

É prerrogativa dos donos do copyright do Linux exigir a obediência aos termos
da GNU GPL para o seu código. A FSF não é um deles; nós nunca estivemos
envolvidos no desenvolvimento do Linux, o kernel. A FSF detém o copyright
de vários outros componentes principais do sistema operacional GNU/Linux,
mas estes programas não estão envolvidos na questão. Portanto a FSF não
é uma das partes desta questão no sentido legal.

Entretanto, nós falamos para Yodaiken que se ele permanecer violando a
GPL, nós podemos decidir suportar os esforços de outras companhias para
invalidar a patente de Yodaiken nas cortes, e nós também podemos suportar
ações tomadas por outros para fazer valer a GPL.

Sobre o GNU/Linux:

GNU é um sistema operacional livre. GNU/Linux é a combinação integrada
do sistema operacional GNU com o kernel Linux.

Chamar o sistema GNU/Linux simplesmente de "Linux" causa confusão.
Tornar consistente a distinção entre o GNU/Linux, o sistema operacional,
e Linux, o kernel, desfaz a confuzão.

Sobre a Fundação para o Software Livre:

A Fundação para o Software Livre, fundada em 1985, é dedicada a
promover os direitos dos usuários de computadores de utilizar, copiar,
modificar e redistribuir programas de computador. A FSF promove o
desenvolvimento e o uso de software livre e de documentação livre para
o software livre. O seu site web é localizado em http://www.gnu.org/.
A sede fica em Boston, MA, EUA.
