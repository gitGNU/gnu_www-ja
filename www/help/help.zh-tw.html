如何幫助 GNU 專案 之指南

如果您想要幫忙 GNU 專案，GNU 志工協調者 <gvc@gnu.org>可以幫助您。 他們會將您與其他對於相似專案有興趣或正從事中的人建立起聯繫。當您從 我們的工作列表中選取了一項工作時，請告知他們你有興趣進行它。

如果您正在進行來自這個列表的任何工作時，請告知GNU志工協調者 <gvc@gnu.org>。 我們想要 記錄並了解 志工們正從事什麼工作。

對於有關GNU專案的一般性問題，請電郵至 gnu@gnu.org ；對於有關網站的問題及建議，請電郵至 webmasters@gnu.org.

如何幫助 GNU 專案

此列表粗略地以緊急、愈需要幫助的項目愈接近上面的方式排序。請注意，在此 列表中的許多項目連結至較大的延伸列表中。

  • ACM Crossroads 雜誌正在尋找一位新的專欄作者 撰寫一系列有關於 自由軟體 和開放原始碼 的短文。理想的申請者應該具備有諸如：各類自由軟體許可證間 之區別、流行的自由套裝軟體...等多方面的知識，以及良好的寫作經驗。如果 您對這份工作有興趣，請電郵至 ACM Crossroads 的主編 Bill Stevenson crossroads@acm.org。完整的申請 作業需要一篇寫作範例（於 2002-1-17 公告）。

  • 我們正在尋找一些可以 以翻譯文件或 howtos 的方式 來幫忙發表 全球各地 有關於 自由軟體在教育上的應用現況 的人。如果有人可以接收並管理整個 GNU 教育 活動，我們會更高興。

  • 如果您很熟悉 Texinfo，您可以重新格式化來自我們 哲學頁面 的某些短文為 Texinfo。

  • 如果您有執行 USENET 新聞 spool 的經驗並能夠連至某一個，您可以再次使 gnu.* 新聞群組成為工作狀態，並且自我們的郵寄名單中設定適當的 gatewaying 至新聞群組。請聯絡 <gvc@gnu.org> 若您能從事這項工作。

  • 幫助編譯 一個已知自由軟體的目錄 之志工。這項工作包括了下載自由軟體套 件，找出有關這些套件的資訊，並且檢查以及驗證這些程式的許可證。

    如果您有興趣自願進行這項工作，請聯絡 <bug-directory@gnu.org>。

  • 撰寫、評估以及使用自由軟體。

    • 安裝並使用 自由軟體。 目前最簡單的方式為安裝並使用 GNU/Linux 系統。

    • 評估自由軟體。
      • 您可以成為一個 GNU 軟體評估小組的成員。

    • 撰寫自由軟體。
      • 下列的 GNU 套件需要維護者（請聯絡 <maintainers@gnu.org> 若您願意成為志工）：
        • GNU dumb
        • The Hyperbole GNU Emacs Package
        • The GNU Accounting Utilities
      • 本 GNU 工作列表 列出了需要志工的新工作。
      • 此 Savannah 的幫助需求 頁面，包含了寄至位於 savannah.gnu.org 的 GNU 專案的幫助需求發布訊息。
      • 此 幫助需求 頁面列出了不是位於 savannah.gnu.org 的主動地尋求更多幫助的已發行 GNU 專案。
      • 此 GNU 專案 頁面列出了主動地尋求更多幫助的正在開發中的GNU軟體。
      撰寫軟體時，請遵照 GNU 程式撰寫標準 ，並當您維護程式時閱讀 給 GNU 軟體維護者的資訊。

      如果您需要某些資源以開發 GNU 軟體，可查閱 GNU 軟體開發者資源 網頁。

  • 撰寫 GNU 軟體之文件，並且使用這些 資源，秘訣和提示。

  • 請以撰寫更好的 GNU 出版書籍之說明來幫忙。

  • 以同意署名至驗證讀者郵寄清單的方式作為驗證讀者的志工 〈您可以藉由傳送一封具有 ``subscribe'' 字眼於本文的郵件到 <proofreaders-request@gnu.org> 來達成。

  • 告知其他人有關GNU專案以及自由軟體基金會的事：

    • 提供您的朋友有關 GNU 哲學 和 軟體 的情報。

    • 告知您的朋友所謂的 "Linux" 作業系統事實上指的是 GNU/Linux： 也就是，GNU 系統再加上 Linux 核心。您只要在說寫有關此系統時能夠 在 GNU/Linux 以及 Linux（核心）之間作個 一致的、清楚分別的 陳述， 就能 幫我們一個大忙。

    • 同時
      增加一個連結至GNU首頁 到您的首頁，
      並請求其他人也跟著做。

    • 同時
      鏡射 本網站，就如同這些已如此做了的 網站 一樣，
      並請求其他人也跟著做。

  • 當您與 不重視自由與社群的 人們談論時， 您可以向他們展示自由軟體事實上的優點 （請到 為什麼 開放源／自由 軟體？ 看看這些數字！ 看些有用的證據）。

  • 以下列方式幫助自由軟體基金會募集資金，讓我們得以 撰寫更多自由軟體與文件：

    • 捐款 給自由軟體基金會。

    • 向自由軟體基金會 訂購 使用手冊、T-恤特別是 CD-ROM。 大部分自由軟體基金會的經費來自於販賣每一個人都可以自由 地複製的東西之複製品。

    • 說服您的辦公室向自由軟體基金會 訂購 使用手冊、T-恤特別是 CD-ROM。

    • 說服您的辦公室向自由軟體基金會訂購 豪華發行套件。

    • 以 販賣自由軟體 並捐出一部分所得給 自由軟體基金會 或其他自由軟體的開發專案。 藉由資助開發工作，您可以讓自由軟體的世界變得更好。

      發行自由軟體是一個為開發工作募集資金的機會。 不要浪費了。

  • 自願去確認來自於我們的 哲學頁面 和其他 GNU 網址之文章位於 並且/或 被連結於 WWW 目錄、入口、和許多 其他的目錄網站，如：Yahoo！、dmoz.org、和 Google。讓這些網站增加有關 我們不同網頁的詳細項目，並確定來自我們的 哲學頁面 和其他 GNU 網址之文章經常得以在適當的分類中被連結到。

    如果您願意幫助我們進行這項工作，請聯絡 GNU 志工協調者 <gvc@gnu.org>。

  • 捐贈辦公耗材及辦公設備給自由軟體基金會。 提供它們到 <fsforder@gnu.org>。

  • 捐贈 硬體 給自由軟體基金會。

  • 進行我們 網站 所需要完成的其中一項 工作 (14k characters)。

  • 如果 您或您的公司 以某種方式支持或發展自由軟體，您可以將您自己〈或您的公司〉 列表於 GNU 服務目錄。

  • 如果您經營的公司需要雇用人來操作自由軟體，您可以將廣告公布在 自由軟體工作頁面。
