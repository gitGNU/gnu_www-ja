帮助 GNU 工程指南

如果您想帮助 GNU 工程，GNU 志愿者协调人员 <gvc@gnu.org> 能协助您。他们将让你同有兴趣或正在进行相同项目的志愿者联系。如果您 已经从我们的任务列表中选中了一个任务，请让他们知道您有志于它。

如果您从任务表中选中了一项任务，请通知 GNU 志愿者协调人员 <gvc@gnu.org>。我们希望记录下志愿 者正在从事的工作。

对于有关 GNU 工程的一般性问题，请发电子邮件至gnu@gnu.org。 对于有关网站问题和建议请发电子邮件至webmasters@gnu.org.

如何帮助 GNU 工程

该列表基本上是依紧急程度排序，前面的项目较紧急。请注意，在此列表中的许 多项目连接至较大的延伸列表中。

  • ACM Crossroads 杂志正在寻找一个专栏作者写一系列关于自由软件和源 代码开放的短文。申请者需要有诸如各类自由软件许可证之间区别，流行的自 由软件包等的知识。同时也要求有较好的写作功底。如果您对这份工作有兴趣， 请发电子邮件至 ACM Crossroads 的主编 Bill Stevenson crossroads@acm.org。提交申请时请附上一篇例文。( 2002-1-17 提交)

  • 我们正在寻求人们帮助我们编写或翻译关于世界范围内自由软件在教育领域的应 用的文档以及HOWTOS。如果有人能管理整个 GNU education活动我们会更高兴。

  • 如果您熟悉 Texinfo, 您可以帮助把我们的一些文章从philosophy page格式化为 Texinfo。

  • 如果您有运行 USENET 新闻组的经验，并且又可以访问一个，您可以帮助 gnu.* newsgroups 再次运行起来，并通过适当的设置，把我们的邮件列表和该新闻组连接起来。 如果您能胜任该任务，请联系 <gvc@gnu.org>

  • 帮助编辑 已知自由软件目录的志愿者。这项任务包括下载自由软件包，搜集关于这些包的信息， 检查和修正程序的许可证。

    如果您有兴趣做这项工作，请联系<bug-directory@gnu.org>.

  • 编写，评估和使用自由软件。

    • 安装和使用 自由软件. 目前最简单的方法是安装并使用GNU/Linux系统。

    • 评估自由软件。
      • 您可以成为一名 GNU 软件评估组的成员。

    • 撰写自由软件。
      • 以下 GNU 软件包需要维护者 (如果您愿意成为志愿者，请联系 <maintainers@gnu.org> ):
        • GNU dumb
        • The Hyperbole GNU Emacs Package
        • The GNU Accounting Utilities
      • 此 GNU 工作列表 列出了需要志愿者的新工作。
      • Savannah 帮助需求 页面列出了 savannah.gnu.org需要的帮助信息。
      • 此帮助需求 列出了除了savannah.gnu.org以外，其他的 GNU 工程寻求帮助的信息
      • 此 GNU 工程 页面列出了正在开发的 GNU 软件，它们正在寻求更多帮助。
      撰写软件时，请遵循 GNU 编码标准 ；维护软件时，请阅读 GNU 软件维护者信息。

      如果您需要开发GNU软件的资源，请浏览 GNU软件开发者资源 网页。

  • 在为 GNU 软件书写文档时,请使用如下 资源，提示，暗示。

  • 请帮助编写更好的 GNU 印刷版书籍描述.

  • 通过加入验证读者邮件组可以成为作为验证读者（您可以通过发一封含有“subscribe”字眼的电子邮件到 <proofreaders-request@gnu.org> 来成为验证读者）。

  • 告知他人有关 GNU 工程和自由软件基金会：

    • 告知您的朋友 GNU 哲学 和 软件.

    • 告知您的朋友 "Linux"操作系统实际上应为 GNU/Linux： GNU 系统，加上 Linux内核。您只要在说写有关此系统时 在GNU/Linux以及 Linux（核心）之间作个一致的、清楚分别的陈述， 就能帮我们一个大忙。 只花一点点时间就可以去掉一个老的习惯。

    • 同时
      加入一个链接至 GNU首页到你的首页，
      并建议其他人也这么做。

    • 同时
      镜像 本站就像 已经做了的网站 一样，
      并建议其他人也这么做。

  • 当您与不重视自由和社区的人们谈论时，您可以向他们展示自由软件的实际优点（请到 为什么使用开放源代码/自由软件？看看这些数字）。

  • 以下列方式帮助自由软件基金会募集资金，让我们得以编写更多的自由软件和文档：

    • 捐款给自由软件基金会。

    • 向自由软件基金会定购手册，T 恤衫特别是 CD-ROM。大部分自由软件基金会的经费来自于出售任何人都可以自由复制的东西的复制品。

    • 说服您的办公室向自由软件基金会定购手册，T 恤衫特别是CD-ROM。

    • 说服您的办公室向自由软件基金会定购豪华套装发布版。

    • 通过出售自由软件 并捐出一部分所得给 自由软件基金会或其他自由软件的开发计划。通过资助开发工作 ，您可以让自由软件世界变得更好。

      发行自由软件是一个为开发工作募集资金的机会，抓住这个机会！

  • 自愿去确认来自于我们的 哲学页面和其他 GNU 网址的文章位于并且/或被链接于WWW目录、入口和许多其他的目录网站，如：Yahoo！、dmoz.org、和 Google。让这些网站增加有关我们不同网页的详细项目，并确定来自我们的哲学页面和其他GNU网址的文章经常得以在适当的分类中被链接。 如果您愿意帮助我们进行这项工作，请联络 GNU 志愿者协调人员 <gvc@gnu.org>。

  • 捐献办公耗材和设备给自由软件基金会，提供它们到 <fsforder@gnu.org>。

  • 捐献硬件给自由软件基金会。

  • 进行我们网站所需要完成的其中一项 工作(14k characters)。

  • 如果您或您的公司以某种方式支持或开发 自由软件，您可以将您自己（或您的公司）列于GNU 服务目录。

  • 如果您经营的公司需要雇佣人来操作自由软件，您可以将广告公布在自由软件工作页面上。
