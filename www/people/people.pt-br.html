Quem é quem no GNU

[ Catalão | Inglês | português do Brasil ]

Aqui temos uma lista em ordem alfabética de alguns contribuidores do GNU. Também temos contribuidores citados em nossas páginas de webmasters e tradutores das páginas em www.gnu.org.

Se você desenvolveu um grande pacote GNU ou realizou bastante trabalho para o Projeto GNU de alguma outra maneira, nós gostaríamos de listar você também. Por favor peça a webmasters@gnu.org para adiciona-lo.

Observe que nossa política é não fazer links para páginas ou sites cujo assunto sejam softwares proprietários, e nós também evitamos fazer links para sites que sejam orientados a vendas no seu tom ou foco. Por favor tenha isso em mente quando escrever o seu pedido.

Se você está procurando por alguém para apresentar uma palestra ou participar em algum evento representando do Projeto GNU ou a Fundação para o Sofware Livre, por favor veja a nossa página de palestrantes. If you are looking in particular for someone to give a speech or participate in an event to represent the GNU Project or the Free Software Movement, please see our speaker page.

Este lugar está reservado para o seu nome, quando você escrever software livre para o Projeto GNU.

Mark Adler
é co-autor do gzip e escreveu o código de descompressão.

Tom Cato Amundsen <tomcato@iname.com>
é o autor do GNU Solfege, um programa para treinamento de audição musical para o GNOME, e ele também realizou algum trabalho com fontes e mudela-book para o GNU Lilypond.
Ele recemente obteve o seu Mestrado em Educação Musical, mas gasta tempo demais programando o Solfege e utilizando software livre.

Karl Berry
esteve envolvido com o GNU desde 1989. Ele foi co-autor dos utilitários GNU para manipulação de fontes, e atualmente trabalha com a FSF como mantenedor voluntário do Texinfo. Ele também realiza uma série de tarefas voluntárias relacionadas com as distribuições do TeX, especialmente o Web2c.

Lorenzo Bettini
é o autor do GNU java2html e do GNU cpp2html: estes programas recebem um fonte em java e C/C++, respectivamente, e produzem uma versão em HTML com sintaxe colorida.

Jim Blandy
tem trabalhado irregularmente com a Fundação para o Software Livre por nove anos. Ele atualmente mantém o Guile como voluntário. Juntamente com Richard Stallman, ele foi responsável pela da versão 19 do GNU Emacs. Jim vive em Bloomington, Indiana.

Paolo Bonzini <bonzini@gnu.org>
foi convertido do mundo Windows para o mundo GNU/Linux depois que ele assumiu a manutenção e o desenvolvimento do GNU Smalltalk. Ele desenvolve software livre no tempo que ele não utiliza estudando e tentando convence os seus amigos do "o fonte é meu" dos benefícios do software livre.

Stephen F. Booth

Daniel Bump

James Craig Burley

Thomas Bushnell, BSG
é o principal arquiteto do GNU Hurd, que é o kernel do sistema GNU. Anteriormente ele manteve o GNU tar e ainda escreveu um interpretador BASIC. Ele também fez muitas outras coisas, algumas delas sem qualquer relação com computadores.

Thomas mudou o seu nome, que era Michael Bushnell, em 1996.

Robert J. Chassell

Igor Támara Patiño

Allin Cottrell

Rick Crelia

Loic Dachary

Ben Elliston
