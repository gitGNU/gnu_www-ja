Por Favor Ajude a Escrever Documentação

Por favor contacte os Coordenadores de Voluntários do GNU para assistência em como iniciar. Por favor também veja escrevendo software livre para listas que incluem projetos que necessitam de alguém que escreva a sua documentação.

Os Padrões de Codificação do GNU incluem uma excelente seção sobre documentação de programas e o Manual do Texinfo contém muitas dicas e macetes. A página sobre licenças contém uma discussão sobre, assim como o texto, da Licença de Documentação Livre do GNU.

Como Obter a Documentação do GNU

A documentação do GNU está disponível de vários modos diferentes. Nossos manuais vem com permissão de serem copiados e modificados. O livro mais antigo que pode ser distribuído livremente é o Sutra do Diamante, publicado na China no ano de 868. Para uma introdução à documentação livre, veja o artigo de Richard Stallman sobre Software Livre e Manuais Livres.

  • Encomende manuais impressos e cartões de referência da FSF.
  • Consulte os Manuais Online.
  • Baixe por FTP de um site próximo de você.
    Os fontes em TeX ou Texinfo da maioria dos manuais do GNU e dos cartões de ferência é incluído com o código-fonte dos programas no mesmo pacote, e não são distribuídos separadamente.
  • Obtenha versões em Postscript ou HTML em sites de Documentação de Software Livre
  • Faça uma cópia xerox da documentação impressa de um amigo.
Se você não adquirir manuais ou cartões de referência da FSF, por favor considere fazer uma doação para nos ajudar a escrever mais documentação.

Descrição dos Manuais que Foram Impressos como Livros

Aqui temos a descrição de todos os manuais do GNU publicados e impressos como livros pela FSF. Veja ISBNs das edições atuais abaixo. Use o Formulário de Pedidos da FSF para comprar livros nas eduções atuais ou em algumas edições antigas, para seu próprio uso. Por favor contate a FSF se você estiver interessado em comprar publicações da FSF para revenda.

Vários manuais do GNU foram traduzidos parao Japonês e estão disponíveis no Japão. Se você souber de outras traduções publicadas como livros, por favor contate a FSF.

Nós necessitamos de voluntários para manter esta informações.
Essas descrições necessitam ser trabalhadas. Cada uma deve ser expandida à partir do texto da contra-capa do manual e/ou do texto na capa sobre-capa, e/ou da introdução de cada manual, e/ou de outros modos úteis.
Cada manual impresso deve ter a sua própria página, incluindo uma imagem reduzida e em tamanho real da capa frontal e da contra-capa; links para o Formulário de Pedidos da FSF, para a página correspondente neste site abaixo de http://www.gnu.org/software/*/ que descreve o software, e talvez para outras páginas. O ideal é que cada manual tenha também uma página de histórico, cobrindo edições antigas.

  • O Manual do GNU Emacs (14a. Edição para a versão 20.7) descreve como editar texto com o GNU Emacs. Ele explica recursos avançados, como o modo outline e expressões regulares para pesquisa; como usar os modos especiais de programação para escrever em linguagens tais como C++ e TeX; como usar o utilitário tags; como compilar e corrigir código; como criar os seus próprios mapeamentos de teclado; e outras customizações elementares.

  • O Manual de Referência do Lisp do GNU Emacs (para a versão 21 do GNU Emacs) e o Manual de Referência do Lisp do GNU Emacs, Edição Japonesa (Revisão Japonesa DRAFT 1.0, da edição em inglês 2.4 para a versão 19.29 do GNU Emacs) cobre esta linguagem de programação profundamente, incluindo tipos de dados, estruturas de controle, funções, macros, tabelas de sintaxe, busca e comparação, modos, janelas, mapeamentos de teclas, compilação em bytes, e a interface com o sistema operacional. O código-fonte está disponível no diretório de FTP ftp://etlport.etl.go.jp/pub/doc/gnu-jp/elisp-manual/.

  • Programando em Emacs Lisp: Uma Introdução (Edição 1.05) para pessoas que não estão necessariamente interessadas em programação, mas que desejam customizar ou extender o seu ambiente computacional. Se você ler este livro no modo Info do Emacs, você poderá executar os exemplos diretamente.

  • O Manual da Calculadora do Emacs (Edição 2.02 para a Versão 2.02) é tanto um tutorial quanto um manual de referência. Ele ensina como realizar aritmética básica, como usar a calculadora para álgebra, cálculo (diferencial e integral) e outras formas de matemática, e como extender a calculadora.

  • Utilizando e Portando o Compilador GNU C (para a Versão 2.95) Ensina como instalar e utilizar o Compilador GNU C e como porta-lo para novos sistemas. Ele lista novos recursos e incompatibilidades do GCC, mas as pessoas que não são familiares com C necessitarão como complemento de uma boa referência da linguagem C. Ele também cobre o G++, o Compilador GNU C++.

  • O Manual de Referência da Biblioteca C do GNU (para a versão 2.x) descreve os recursos da biblioteca, incluindo tanto o que o Unix chama de "funções da biblioteca" quando as "chamadas ao sistema". Ele foi publicado como um conjunto de dois volumes totalizando mais de 1100 páginas! Por favor enviem correções para bug-glibc-manual@gnu.org. O manual não cobre as bibliotecas do C++.

  • Depuração com o GDB (para a versão 5.0) explica como executar o seu programa sob o controle do Depurador do GNU, examinar e modificar dados, modificar o fluxo de controle de um programa, e usar o GDB através do GNU Emacs.

  • GNU Make (para a versão 3.79) descreve o GNU make, um programa utilizado para construir partes de outros programas. O manual ensina como escrever "makefiles", que especificam como um programa deve ser compilado e como os seus arquivos dependem uns dos outros. Está incluído um capítulo introdutório para usuários iniciantes e uma seção sobre dependências geradas automaticamente.

  • O Manual do Bison (Edição de Novembro de 1999 para a versão 1.29) ensina como escrever gramáticas livres de contexto para serem convertidas pelo programa Bison em analisadores sintáticos escritos em C. Não é necessário ter conhecimento anterior sobre geradores de analisadores sintáticos.

  • O Manual do Flex (Edição 1.03 para a Versão 2.3.7) ensina como escrever um analisador léxico para que será convertido pelo programa flex em um analisador léxico escrito em C ou C++ que reconhece os padrões definidos. Você não necessita de conhecimento prévio sobre analisadores léxicos.

  • GAWK: O Manual do Usuário do GNU Awk (Segunda Edição para o GAWK Versão 3) ensina como utilizar o GAWK. Ele é escrito para aqueles que nunca haviam utilizado o awk e descreve os recursos desta proderosa linguagem de manipulação de strings e registros.

  • Texinfo: O Formato da Documentação do GNU (para a Versão 4) explica a linguagem de marcadores que produz a nossa documentação on-line e as suas cópias impressas. Ele ensina como criar tabelas, listas, capítulos, nós, índices, referências cruzadas e como localizar erros.

  • O Manual do Termcap (Terceria Edição, revisada, para a versão 1.3), é frequentemente descrito como "duas vezes todo o que você sempre quis saber sobre o banco de dados termcap", as definições das capacidades do terminal, e o processo de interrogar a descrição de um terminal. Este manual é voltado principalmente para programadores.
