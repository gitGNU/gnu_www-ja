フリーソフトウェア
フリーソフトウェアとは、すべての人に対して改変や代価の有無を問 わず利用、複製、頒布する許可が与えられているソフトウェアのこと であり、ソースコードが入手可能であることが必須とされています。 「ソース無くしてソフトウェア無し」というのが簡略な定義です。完全な定義もご参照くださ い。

「free software」 という言葉の各国語訳もご用意しています。

もしプログラムがフリーであれば、GNUやフリーのGNU/Linux システムのよう なオペレーティングシステムのフリーなバージョンに含めることがで きます。

プログラムをフリーにするにはいくつもの異なった流儀があります。 すなわち、選択肢としては複数選べるが、そのどれを選んでもなおプ ログラムをフリーと出来るような細かい論点が多くあるということで す。可能なバリエーションのいくつかについては後述します。

フリーソフトウェアでは自由が問題となるのであって、価格は問題で はありません。一方独占的なソフトウェア企業は、往々にして価格に ついて言及する際「フリーソフトウェア」という用語を使います。そ ういう場合、彼らが意味しているのはバイナリのコピーが無料で1つ手 に入るということだったり、あなたが買おうとしているコンピュータ におまけで1コピー付いてくるということだったりします。こういった ことは、私たちがGNU プロジェクトにおいて「フリーソフトウェア」 で意味していることとは何の関係もありません。

このように意味をはき違えている可能性がありますので、ソフトウェ ア企業が自分たちの製品をフリーソフトウェアだと言っている場合に は、いつでも実際の頒布条件をチェックして、ユーザにとって本当に フリーソフトウェアという語が含意するすべての自由を保障されてい るかどうか確かめるようにして下さい。その製品は本当にフリーソフ トウェアなのかも知れませんが、そうでないこともあるのです。

多くの言語では、自由の「フリー」と無料の「フリー」を二つの別の 単語を使って表現し分けています。たとえば、フランス語では「libre」 と「gratuit」といった具合です。英語にも「gratis」という語があり、 これを使えば価格について言及していることは誤解の余地無く示せる のですが、一般的な形容詞で自由という意味での「フリー」を過不足 なく表現できるものが存在しないのです。そのような語があれば説明 する際に役に立つのですが、これは不運なことでした。

フリーソフトウェアは多くの場合、フリーではないソフトウェアより も信頼できます。

オープンソースソフトウェア
「オープンソース」ソフトウェアという用語は、大体においてフリーソ フトウェアと同じことを意味するために一部の人びとが使っていること ばです。 私たちは「 フリーソフトウェア」という語の方を好んでいます。リンクを辿っ て理由をお読みください。

パブリックドメインソフトウェア
パブリックドメインソフトウェアとは、著作権が放棄されたソフトウェ アのことです。これはコピーレフト が主張されないフリーソフトウェアの特別なケースに相当し、この ソフトウェアの特定の複製物や改変されたバージョンは全くフリーで無 くなってしまう可能性があるということを意味します。

「パブリックドメイン」という用語を、「フリー」とか「無料で 入手可能」というような意味を軽くにおわせるつもりで使う人びとがい ます。しかしながら、「パブリックドメイン」というのは法律用語の一 つであり、厳密に「著作権が放棄されている」ということしか意味しな いのです。意味をはっきりさせるために、私たちは本当に著作権が放棄 されていることを示したいときのみ「パブリックドメイン」という語を 使い、他の意味を伝えたいときは他の用語を使うことをお勧めします。

コピーレフトが主張される(Copylefted)ソフトウェア
コピーレフトが主張されるソフトウェアとは、その頒布条件において、 再頒布者がソフトウェアの再頒布や改変をする場合にいかなる追加的制 限を加えることも許していないフリーソフトウェアのことです。つまり、 そのソフトウェアのすべての複製物が、たとえ改変されていたとしても、 フリーソフトウェアでなければならないということを意味します。

GNU プロジェクトでは、私たちが書くほとんどすべてのソフトウェアに コピーレフトを主張しています。なぜなら私たちの目標はすべての ユーザに「フリーソフトウェア」ということばが含意する自由を与 えることだからです。コピーレフトがどのように機能するか、そしてな ぜ私たちがそれを適用するかについての詳細はコピーレフトの主張をお読み ください。

コピーレフトは普遍的な概念です。実際にプログラムにコピーレフトを 主張するには、特定の頒布条件のセットを適用する必要があり、またコ ピーレフトな頒布条件を書くには多くの可能なやり方があります。

コピーレフトが 主張されない(Non-copylefted)フリーソフトウェア
コピーレフトが主張されないフリーソフトウェアには作者から再頒布と 改変の許可が与えられていますが、同時に再頒布や改変にあたって制限 を追加することも許可されています。

もしプログラムがフリーでも、コピーレフトが主張されていなければそ のいくつかの複製物や改変されたバージョンは全くフリーでなくなるか もしれません。改変されていようがいまいが、ソフトウェア企業はその プログラムをコンパイルして、実行形式ファイルを独占的なソフト ウェア製品として頒布できるのです。

X Window Systemがその好例です。X コンソーシアムはX11を、コピーレフトを主張しないフリーソフトウェア とする頒布条件の下で発表しています。その気さえあれば、あなたはそ ういう頒布条件を持つ複製物を入手できますし、それはフリーです。し かし、世の中にはフリーではないバージョンも存在し、また人気のある ワークステーションやPCのグラフィックボード上ではそういったフリー ではないバージョンしか動作しないということがあるのです。もしあな たがこのようなハードウェアを使っているならば、あなたにとってX11 はフリーソフトウェアではありません。

GPLで保護された(GPL-covered)ソフトウェア
GNU GPL (一般公衆利用許諾契約書) (20k キャラクタ) はプログラムにコピーレフトを主張するための頒 布条件の、具体的なセットの一つです。GNUプロジェクトではほとんどの GNU ソフトウェアの配布条件としてこれを使っています。

GNU システム
GNU システムはフリーの完全 なオペレーティングシステムで、Unixに似ています。

Unixライクなオペレーティングシステムは多くのプログラムで構成され ます。GNU システムはすべての GNU ソフトウェアに加え、X Window SystemやTeX といった GNU ソフトウェアではない多くのソフトウェアパッ ケージも含んでいます。

私たちは1984年以来このシステム向けの様々なコンポーネントを開発・ 集積し続けており、「完全なGNUシステム」の最初のテストリリースは 1996年に行われました。2001年現在、GNUシステムは十分信頼に足る動作 を見せており、また人々はGNOMEやpppがこのシステム上で動くよう作業 しています。一方、GNUシステムの派生物の一つでLinuxをカーネルとし て利用している GNU/Linuxシステム は非常な成功を収めました。

GNUの目的はフリーであることですから、GNUシステムのどの一部分をとっ てもフリーソフトウェアでなければなりません。しかしそれらすべてに コピーレフトが主張されている必要はないのです。そのプログラムが技 術的な目標を満たすのに有用ならば、どの種のフリーソフトウェアも GNU に含めるのに適しますし、かつ合法です。私たちは X Window Systemのようなコピーレフトが主張されていないフリーソフトウェアを 利用することができますし、実際利用しています。

GNU プログラム
「GNU プログラム」はGNU ソフトウェアと同じ意味です。あるプログラムFooがGNUソフトウェ アならそれはGNUプログラムです。

GNU ソフトウェア
GNUソフトウェアはGNUプロジェクトの後援を受けて 発表されたソフトウェアです。多くのGNUソフトウェアにはコピーレフトが主張されてい ますが、すべてではありません。しかし、すべてのGNU ソフトウェアは フリーソフトウェアでな ければなりません。

もしそのプログラムがGNU ソフトウェアならば、それをGNU プログラム とも呼びます。

GNU ソフトウェアのいくつかはフリーソフ トウェア財団のスタッフによっ て書かれましたが、多くは ボランティ アによって寄与されたものです。GNU ソフトウェアのいくつかはフ リーソフトウェア財団に著作権があります。他のものについてはそれを 書いた寄与者に著作権が帰属します。

疑似フリー(Semi-free)ソフトウェア
疑似フリーソフトウェアとは、フリーではないが、非営利目的であれば 個人の利用や複製、頒布、改変(改変されたバージョンの頒布を含む)を 許可しているソフトウェアのことです。PGPが疑似フリーなプログラムの 一例です。

疑似フリーなソフトウェアは 独占的ソフトウェ アよりずっとましですが、依然としていくつかの問題を引き起こし ます。よって私たちはそれをフリーなオペレーティングシステムで使う ことはできません。

コピーレフトが課す制限は、ユーザにとって不可欠な自由をすべてのユー ザに対して保護するために用意されたものです。私たちにとって、プロ グラムの使用に実質的な制限を課す上での唯一の正当な理由は、他の人 びとが制限を追加することを防ぐということ以外にありません。一方疑 似フリーなプログラムには、まったく自己中心的な目的を達成するため に追加された制限が課せられています。

フリーなオペレーティングシステムに疑似フリーのソフトウェアを含め ることは不可能です。それは、オペレーティングシステム全体としての 頒布条件がそれに含まれるすべてのプログラムの頒布条件を総合したも のになるからです。一つ疑似フリーなプログラムをシステムに加えると いうことは、システムを全体として疑似フリーにしてしまうだ けのことに他なりません。 こんなことになって欲しくない二つの理由があります。

  • 私たちは、フリーソフトウェアはすべての人のためであるべきだ と考えています。ですから、対象は学校やホビイストだけに留まらず、 ビジネスの世界も含まれます。私たちとしてはビジネスの世界の人び とにも全体としてのGNU システムを使うようお勧めしたいので、私た ちは疑似フリーなプログラムを含めるべきではないのです。

  • Linux をベースとした GNU システムを含むフリーなオペレーティングシステムの商業的 再頒布は非常に重要です。ユーザは商業的に頒布されるCD-ROMを買う ことができるのを喜びます。一つ疑似フリーなプログラムをオペレー ティングシステムの中に含めば、そのために商業的なCD-ROM頒布を中 止しなければならなくなるでしょう。

フリーソフトウェア財団そのものは非営利的な団体ですから、私たちが 「内部的」に疑似フリーなプログラムを使うことは法的に許可されてい ると考えられます。しかし私たちは、そのようなプログラムを使いませ ん。そのことが、GNU の中にも含めることができるプログラムを手に入 れようとする私たちの努力を土台から侵食するだろうからです。

もし疑似フリーなソフトウェアを使って行う必要のある作業があれば、 私たちがそれをこなすフリーのプログラムを手に入れるまで、GNU シス テムには欠落があるということになります。私たちはボランティアの皆 さんに、「私たちはまだ、 GNU の中にこの作業をこなせるプログラムを 有していません。ですからそういうプログラムを皆さんが書いてくださ ることを望みます」と言わなければなりません。もし私たち自身がそう いった仕事をするのに疑似フリーなプログラムを使っていたら、言行不 一致の誹りを免れないでしょう。そんなことをすれば、フリーな代替品 を書こうとする私たちや私たちの見方に共鳴してくださる他の方々のや る気を削ぐことになってしまいます。ですから私たちは疑似フリーソフ トウェアを使いません。

独占的(Proprietary)ソフトウェア
独占的(プロプライエタリ)ソフトウェアは、フリーでもなければ疑似フ リーでもないソフトウェアのことです。その利用、再頒布、改変は禁止 されているか、許可を得ることが必要とされているか、あるいは厳しい 制限が課せられていて自由にそうすることが事実上できなくなっていま す。

フリーソフトウェア財団では、そのプログラムのフリーな代替品を書く という特定の目的のために一時的にそうせざるを得ない場合を除き、い かなる独占的プログラムをも私たちのコンピュータにはインストールし ないという規則を守っています。それ以外には、独占的プログラムをイ ンストールするいかなる可能な言い訳も存在しないと思います。

例えば、1980年代においてはUnixを私たちのコンピュータにインストー ルすることは正当化しうると考えていました。なぜなら、私たちはそれ を Unix のフリーな代替品を書くために使っていたからです。現在では フリーなオペレーティングシステムが入手可能となっていますから、こ の言い訳はもはや通用しません。私たちはフリーではないオペレーティ ングシステムをすべて除去し、新しく導入したコンピュータではすべて 完全にフリーなオペレーティングシステムを動かさなければならないと いうことにしました。

GNUのユーザやGNUに対する寄与者の皆さんにまで、このルールに従って 生きることを強制するつもりはありません。これは私たちが自分たち自 身に課した規則です。しかし、私たちはあなたもこれに従ってくださる ようになることを望んでいます。

フリーウェア
「フリーウェア」という用語に、一般的に認められたはっきりとした定 義はありません。しかし多くの場合このことばは、再頒布は許可してい るが改変は許可されていない(そしてソースコードも入手できない)ソフ トウェアパッケージを指すのに使われています。こういったソフトウェ アパッケージはフリーソフトウェアではありません。ですから、 フリーソフトウェアについて言及するのに「フリーウェア」という語は 使わないようお願いします。

シェアウェア
シェアウェアとは人びとが複製物を再頒布することは許可しているが、 そのコピーを使用し続ける人は皆ライセンス料を払わなければなら ないと主張するソフトウェアのことです。

シェアウェアはフリーソフトウェアではありませんし、疑似フリーです らありません。それには二つ理由があります。

  • ほとんどのシェアウェアでは、ソースコードが入手できません。 よって、そのプログラムを改変することは全く不可能です。

  • シェアウェアでは、ライセンス料を払わない限りソフトウェアを 複製したりインストールしたりする許可を与えられません。非営利な 活動に参画している個人にすらそうです(実際のところ、皆そんな条件 など無視して適当にそういうことをやっているものですが、ともかく 頒布条件では認めていません)。

商用ソフトウェア
商用ソフトウェアはそのソフトウェアを利用することから利益を得るこ とを目的とする企業によって開発されたソフトウェアのことです。「商 用」と「独占的」とは同じ意味ではありません! ほとんどの商用ソフト ウェアは独占的 ですが、商用のフリーソフトウェアはありますし、フリーではない 非商用のソフトウェアも存在します。

例えば、GNU Adaは常にGNU GPLが指定する条件のもとで配布されており、 その複製物はすべてフリーソフトウェアです。一方でGNU Adaの開発者た ちはサポート契約を販売しています。彼らのセールスマンが顧客になっ てくれそうな人なり企業なりの相手をするとき、こんな風に言われるか も知れません。「商用のコンパイラの方が安心みたいな気がするよ。」 こういう場合セールスマンは、「GNU Ada は商用コンパイラで すよ。たまたまフリーソフトウェアなだけで。」と答えるでしょう。

GNU プロジェクトにとって、重要視する優先順位が違うのです。重要な のは GNU Adaがフリーソフトウェアだということで、それが商用だとか そうではないとかは重要な問題ではないのです。しかしながら、 GNU Ada が商用であるがゆえにその開発に何かプラスがあるならば、それは もちろん有益なことです。

商用のフリーソフトウェアが存在可能であるということに対する認知を 広めることにどうかご協力下さい。「独占的」と意味したいときに「商 用」と言わないように努力することでも助けになります。
