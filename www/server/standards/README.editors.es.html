Pautas para la creación de páginas web en www.gnu.org

Esta es una guía general para todo aquel que quiera escribir una página web para el servidor del proyecto GNU, www.gnu.org. Hay algunas otras guías importantes, una de las más importantes es FSF HTML Style Sheet en http://www.gnu.org/server/fsf-html-style-sheet.html. En http://www.gnu.org/server/standards/readme_index.es.html hay una relación del resto.

Ninguna página del servidor web del proyecto GNU debería hacer referencia a software que no sea libre ni a documentación que tampoco sea libre. Esto es MUY importante.

http://www.gnu.org/boilerplate.html es el patrón usado para todas nuestras páginas. A la hora de escribir una página web, un buen punto de partida es utilizar dicho patrón y seguir las instrucciones incluidas en él. Al hacerlo, elimine los comentarios que no sean necesarios para su página. Este uso del patrón facilita la labor a los futuros encargados de mantener el fuente HTML. También convendría que verificase la posible existencia de versiones especializadas de boilerplate.html en el directorio actual, así como en aquellos que se encuentran por encima de él.

Cuando escribimos una página para un determinado programa queremos que contenga cierta información básica:

  • Una descripción de lo que hace el programa.
  • Dirección desde la que se puede descargar el programa.
  • Dirección a la que dirigirnos para notificar errores.
  • Preguntas frecuentes (FAQs) y documentación, en caso de que esté disponible.
  • Instrucciones de instalación.

La descripción debería ser lo suficientemente extensa como para que el lector pueda hacerse una idea del programa en su conjunto, pero no tan larga como para aburrirle o desanimarle. Si un paquete incluye varios programas pequeños, lo mejor es escribir una descripción general del tipo de programas que contiene el paquete y, a continuación, una breve descripción de cada uno de ellos.

El proyecto GNU pretende albergar todas las páginas web sobre un programa, por lo que si las páginas principales se encuentran en la actualidad en algún otro servidor, considere la posibilidad de trasladarlas al servidor web del proyecto GNU.

Cuando usted mantenga un conjunto de páginas web, quizá para un programa en particular, en www.gnu.org, debería asegurarse de que su nombre y su dirección de correo electrónico figuren en la lista ~www/GNU.web-folk.html en gnudist. Así, los administradores del sitio web sabrán con quién ponerse en contacto si necesitan hacer alguna modificación en las páginas que usted mantiene. Pida en webmasters@gnu.org que le añadan a dicha lista si no puede hacerlo usted mismo. (Tenga presente que no publicamos ~www/GNU.web-folk.html en este sitio web porque algunos voluntarios no quieren que su dirección de correo electrónico pueda quedar expuesta al envío de correo no deseado.)
