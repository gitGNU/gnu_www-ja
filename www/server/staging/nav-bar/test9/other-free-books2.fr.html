Livres libres d'autres éditeurs

Introduction

Inscrivez-vous à notre liste de diffusion (en anglais) concernant les dangers des livres électroniques.

Nous soutenons depuis longtemps la publication de documentation libre (libre dans le sens de liberté, nous ne voulons pas dire gratuit). Les logiciels libres ont besoin de documentation libre car on doit pouvoir copier, modifier et redistribuer la documentation avec les logiciels. Pour être libre, la documentation doit être publiée sous une licence de documentation libre.

Nous publions nous-mêmes un certain nombre de livres de documentation libres. Vous pouvez en commander des exemplaires imprimés, les télécharger à partir du répertoire /gnu/Manuals de notre site FTP et les consulter en ligne, ceci depuis 1986 environ.

Plus récemment, certain éditeurs commerciaux ont commencé à publier des livres libres (tout comme les nôtres, ceux-ci sont libres dans le sens de liberté ; les exemplaires imprimés sont vendus). Cette page référence ceux dont nous avons connaissance. Si vous trouvez un livre libre qui n'est pas indiqué ici, veuillez nous le faire savoir en nous envoyant un courriel à <free-books@gnu.org>.

Note sur Advanced Linux Programming, 1re édition :
L'édition de juin 2001 avait un avis de licence incorrect. La version imprimée indiquait qu'il s'agissait d'un livre privateur. L'éditeur nous a assuré que c'était seulement une erreur d'impression et que la licence était en fait l'« Open Publication License, version 1.0, sans option exercée ». La page « Errata » l'atteste. Nous avons reçu l'assurance que l'avis de licence serait corrigé dans la prochaine édition imprimée.
Note sur Linux Device Drivers, 2e édition :
La version imprimée a incorrectement appliqué la GNU FDL : il manque une copie de la GNU FDL et l'avis de licence est incomplet. O'Reilly corrigera ces problèmes dans la prochaine édition imprimée.
