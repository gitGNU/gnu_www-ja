Consejos para la utilización de duplicados («mirrors»)

Duplicar el servidor FTP de GNU

Antes que nada, si duplica ftp.gnu.org, por favor háganoslo saber enviando un mensaje a webmasters@gnu.org para agregarle a la lista de duplicados. Gracias.

Para duplicar el servidor FTP de GNU por completo se necesitan aproximadamente 40GB de espacio en disco (a 4 de enero de 2007), los cuales están distribuidos de la siguiente manera:

 /gnu:                12G 
 /non-gnu:           300M 
 /old-gnu:           700M 
 /gnu+linux-distros:  27G

Debido a los recursos y problemas de seguridad, ya no ofrecemos servicios de rsync o FTP para actualizar los duplicados directamente. Si usted desea utilizar rsync (lo cual es recomendable) o FTP, existen algunos sitios de terceros confiables que ofrecen acceso al material de ftp.gnu.org. Ellos actualizan directamente de nosotros con regularidad y y usted puede acceder a ellos (mediante rsync) con:

  • rsync://mirrors.ibiblio.org/pub/gnu/ftp/ (EE.UU.)
  • rsync://ftp.funet.fi/ftp/pub/mirrors/ftp.gnu.org/pub/gnu (Finlandia)
  • rsync://gnu.c2sl.fpr.br/gnu/ftp/ (Brasil)

y vía ftp:

  • ftp://mirrors.ibiblio.org/pub/mirrors/gnu/ftp/
  • ftp://ftp.funet.fi/ftp/pub/mirrors/ftp.gnu.org/pub/gnu/
  • ftp://gnu.c3sl.ufpr.br/gnu/ftp/

Si su sitio también puede ofrecer rsync para la duplicación, por favor contáctenos.

Consejos para actualizar desde Ibiblio

  • Ibiblio recomienda usar -rltp --delete para copiar el contenido de ftp.ibiblio.org vía rsync. Usar -x puede causar problemas del lado del cliente porque deshabilitan la compresión. Esas opciones también son útiles para otras fuentes.
  • Para los administradores de duplicados, «ibiblio-mirrors» es una lista de correo, con poco tráfico, para los sitios que duplican el contenido de los servidores de ibiblio. Este es el enlace a la página de la lista de correo: http://lists.ibiblio.org/mailman/listinfo/ibiblio-mirrors
  • Puede enviar preguntas específicas e informar sobre problemas a: help-arroba-ibiblio-punto-org.

Duplicar el servidor FTP Alfa de GNU

Estos sitios también ofrecen alpha.gnu.org para su comodidad al duplicar:

  • rsync://mirrors.ibiblio.org/pub/gnu/alpha/
  • rsync://ftp.funet.fi/ftp/pub/mirrors/alpha.gnu.org/gnu/
  • rsync://gnu.c3sl.ufpr.br/gnu/alpha/
  • ftp://mirrors.ibiblio.org/pub/mirrors/gnu/alpha/
  • ftp://ftp.funet.fi/ftp/pub/mirrors/alpha.gnu.org/gnu/
  • ftp://gnu.c3sl.ufpr.br/gnu/alpha/

Duplicar el servidor web de GNU

Ya no recomendamos la creación de duplicados web, debido a que: (a) actualmente pocos usuarios hacen uso de ellos. (b)muchos duplicados proporcionan información desactualizada o quedan fuera de servicio con relativa frecuencia. Nosotros no queremos que se divulgue información desactualizada acerca del proyecto GNU.

No obstante, si usted desea duplicar www.gnu.org para su uso propio, para nosotros está bien. Le recomendamos que utilice mod_proxy. Aquí tiene unas indicaciones de directivas de Apache para su «VirtualHost»:

ServerName gnumirror.ejemplo.org
CacheRoot "/var/cache/mod_proxy"
CacheDefaultExpire 86400 # para Apache 2.0+, el valor es en segundos
ProxyPass / http://www.gnu.org/
CacheEnable disk /

Le damos las gracias a Juri Hoerner por proporcionarnos esta información de Apache

Duplicar las versiones nongnu publicadas de Savannah

Estamos buscando voluntarios para duplicar las versiones liberadas los proyectos nongnu alojados en savannah.nongnu.org. El total es actualmente de alrededor de 5GB. Para hacerlo, le sugerimos enfáticamente que obtenga y actualice los ficheros usando rsync -aHS --delete-excluded rsync://dl.sv.gnu.org/releases/  /sudirectorio
Las opciones aseguran que sea un duplicado exacto. (Seguramente habrán algunos errores de permisos, ignórelos. Ellos se deben a opciones de configuración o errores por parte de los encargados de mantenimiento del proyecto savannah)

Si puede ayudarnos con ello, favor envíenos un correo electrónico con la url que está duplicando para agregarla a la lista. ¡Gracias!

Por favor, envíe sus comentarios y preguntas sobre la FSF y el proyecto GNU a gnu@gnu.org. También puede contactar con la FSF por otros medios.
Por favor, envíe enlaces rotos y otras correcciones, o sugerencias, a webmasters@gnu.org.
