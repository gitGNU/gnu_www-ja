0. Introduccion:

Despues de responder la pregunta de "... como me conecto a imternet con linux? ..." trillones de veces en el IRC, las news y fido, me he decidido a crear este micro-documento para conectarse a internet con cualkier distribucion.

Kitaos de la cabeza las X, las KDE, las utilidades de configuracion ... lo mas sencillo, efectivo y facil de entender es hacerlo a mano creedme, si no, leed lo ke viene continuacion y vereis:

1. Rekisitos:

Atencion: Es muy importante cumplir todos los rekisitos, no sigas leyendo hasta estar seguro de tener todo lo necesario o no conseguiras conectar.

  • Modem instalado y bien configurado, si no estas seguro leete el modem.htm. Puedes comprobar si el modem esta bien instalado usando un programa de terminal como el minicom.
  • Soporte PPP: consta de dos partes:
    • Soporte ppp en el kernel: para saber si tienes soporte ppp en el kernel escribe dmesg si no ves ninguna referencia al ppp es ke tu kernel no lo soporta, de todas formas puede ke lo tengas como modulo, en este caso haz ls -C /lib/modules/`uname -r`/net, si no aparece algo como ppp.o es ke tampoco lo tienes como modulo. Tu unica solucion es recompilar el kernel con soporte ppp, si no sabes como hacerlo te aconsejo leer el kernel-como.html.
    • Demonio pppd: este es el programa ke se encarga de realizar la conexion. Para saber si lo tienes instalado pulsa pppd, si el resultado es:

      [root:/]# pppd
      bash: pppd: command not found

      es ke no lo tienes; es imprescindible ke lo consigas, busca la version 2.3.5 pues es la ke he usado para hacer este documento.
      Para saber ke version tienes escribe:

      [root:/]# pppd --version
      pppd version 2.3 patch level 5

      Si no sabes de donde sacarlo seguramente lo puedes encontrar en tu CD de linux, o en el ftp de tu distribucion.

  • Una cuenta de acceso a internet mediante infovia
  • Cualkier editor de textos (vi, joe ... incluso cat vale) para editar los archivos de configuracion.

2. Configuracion:

Atencion: El texto entre parentesis son solo aclaraciones, no se os ocurra ponerlo en vuestros archivos de configuracion.

  • Archivo /etc/resolv.conf
    domain DOMINIO_DE_TU_ISP (por ejemplo arrakis.es)
    nameserver DNS_PRIMARIA (por ejemplo 194.224.52.4)

  • Archivo /etc/host.conf
    order hosts,bind
    multi on

  • Archivo /etc/ppp/options
    connect /etc/ppp/infovia
    crtscts
    modem
    passive
    user usuario@servidor (por ejmplo pepe@arrakis)
    noipdefault
    :192.168.0.2
    ipcp-accept-remote
    debug
    defaultroute
    asyncmap a0000
    /dev/modem (o el dispositivo donde esta tu modem, /dev/ttyS?)
    115200

  • Archivo /etc/ppp/infovia (este archivo ha de ser ejecutable, pon chmod +x infovia)
    #!/bin/sh
    /usr/sbin/chat -v "" atdt055 CONNECT ""

  • Archivo /etc/ppp/pap-secrets
    "usuario@servidor" * "password" (las comillas son necesarias, los espacios son tabuladores)

3. Conectando:

  • pppd: establece la conexion

  • pppd: finaliza la conexion (si, la misma orden inicia y finaliza la conexion)

  • tail -f /var/log/messages: informa de lo ke ocurre durante el proceso de conexion, ideal para resolver fallos. Este archivo es el ke debes mirar cuando alguien te pregunta: ... y ke te pone en los logs?

4. Infovia+:

Los cambios necesarios para acceder a Infovia+ son:

  • Archivo /etc/ppp/options: substituir
    user usuario@servidor (por ejmplo pepe@arrakis)
    por
    name usuario@servidor

  • Archivo /etc/ppp/chap-secrets: eliminar todo lo ke haya y poner en su lugar
    usuario@servidor ppp0 password

  • Archivo /etc/ppp/infovia: susbtituir en la linea
    /usr/sbin/chat -v "" atdt055 CONNECT ""
    El 055 por el nodo local de Infovia+

Y ya esta, asi de facil, ahora ya no tienes excusa para no conectarte desde linux!
A partir de aki ya podeis empezar a experimentar con otras cosas, como kppp, acceso a internet de usuarios, seguridad ... pero todo esto desde linux.
